Java Web的案例

回顾

  • 请求响应模型（浏览器发送一个请求，服务器响应一个结果）
  • 浏览器发送请求的方式
    • 地址栏
    • a标签，超链接
    • form表单
  • 请求参数：发送请求时携带的额外的信息
  • 请求发送后，通过URL的IP:端口找到对应电脑的对应服务器软件来处理请求（Tomcat）
    • Tomcat默认的端口是8080
  • Tomcat接收到请求后，会分析URL，找到对应项目下的web.xml文件，查询对应的请求是否有对应的类来进行处理。找到对应的类就会调用对应的Servlet类来处理请求
  • Servlet类必须继承HttpServlet类，重写doGet和doPost方法
  • Tomcat会根据发送请求时，请求参数发送的方式是get还是post分别调用doGet和doPost方法处理请求
  • 处理请求的步骤：
    • 获取请求参数
    • 调用service、dao、操作数据库
    • 共享数据
    • 响应
  • 共享数据通过作用域对象来进行共享
    • request：一次请求响应内有效
    • session：一次会话内有效（浏览器打开进入网站到浏览器关闭，服务器关闭，最后一次请求响应超过30分钟代表一次会话结束）
    • application：整个应用有效（项目发布期间都有效，如果服务器重启，application会重置）
  • 响应
    • 重定向：浏览器发送两次请求，第一次发送请求后，服务器会把结果资源的地址告诉浏览器，浏览器再次发送请求请求对应的结果资源。
    • 请求转发：浏览器发送一次请求，服务器接收到请求后，找到请求对应的资源的路径，会直接把对应的资源文件发送给浏览器。
  • 有时我们需要查询数据库的数据发送到浏览器显示，这些数据是动态变化的。因此不能使用HTML页面来显示，所以提供了JSP页面可以动态的获取从数据库查询的数据并且显示。
  • EL表达式是从作用域对象中获取从数据库中查询出来的数据，然后在JSP页面显示。
    • 基本数据类型和字符串：${key}
    • 实体类：${key.属性｝
    • 集合的：${key[下标].属性}
  • JSTL标准标签库：用来格式化JSP页面中的格式。本质上是自定义的标签，这些标签可以代表特定的含义。
    • 核心标签库
      • c:if
      • c:forEach
    • 格式化标签库
      • fmt:formatDate
